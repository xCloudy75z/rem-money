## Architecture — read this before changing anything

### Module pattern (LOCKED)
Every `src/*.js` is a single IIFE assigning to a global:
```javascript
var ModuleName = (function () {
  'use strict';
  // ...
  return { publicFn };
})();
```
Build script (`scripts/build.js`) concatenates all `src/*.js` between `<!-- MODULES:START -->` / `<!-- MODULES:END -->` markers in `index.html`, inlines `src/styles/main.css` between `<!-- STYLES:START -->` / `<!-- STYLES:END -->`, and produces a single `dist/index.html`. **No bundler, no transpiler.** This is intentional — survivability over modernity.

### Pure-module discipline (LOCKED — enforced by lint)
These modules **MUST NOT** call `new Date()`, `Date.now()`, `Math.random()`, or `crypto.randomUUID()`:
- `src/calc.js`
- `src/store.js`
- `src/validate.js`
- `src/migrate.js`
- `src/seed.js`
- `src/format.js`

Time and IDs are **passed in as arguments**. `scripts/lint-pure.js` greps for violations and fails CI. **Don't disable this.** It eliminates two whole categories of bugs (timezone, flaky tests).

Only `src/app.js` (the orchestrator) is allowed to call those APIs.

## Mobile UX patterns (HARD-LEARNED this session)

### iPhone status bar overlap
- Both `.app-header` AND `.sheet` need to account for `env(safe-area-inset-top)`
- For headers: `padding-top: calc(16px + env(safe-area-inset-top))`
- For sheets: `max-height: calc(100vh - env(safe-area-inset-top) - 24px)` so sheet doesn't extend under status bar
- iOS PWA in standalone mode + `apple-mobile-web-app-status-bar-style: black-translucent` makes content render UNDER status bar by default

### iOS white-icon problem
SVG-only icons appear as **white squares** on iPhone home screen — iOS Safari doesn't render SVG `apple-touch-icon` reliably. You **must** ship PNG fallbacks:
- `apple-touch-icon-180.png` (180×180) — for iOS
- `icon-192.png` (192×192) — for Android home screen
- `icon-512.png` (512×512) — for Android splash
- `icon-512-maskable.png` (512×512 with safe-area padding) — Android adaptive

The PNGs were generated this session via `preview_eval`: render SVG to canvas → toDataURL → write bytes via PowerShell. See git history for the exact one-shot command, or use `tools/generate-icons.html` (a Canvas-based generator that downloads all 4 PNGs).

### Sheet anatomy (current pattern)
Every sheet uses the `Sheet` controller in `src/components/sheet.js`:
```html
<div class="sheet">
  <div class="sheet-chrome">                  <!-- sticky at top:0 -->
    <div class="sheet-handle"></div>          <!-- 4px drag handle, centered -->
    <div class="sheet-header">                <!-- flex row -->
      <h2 class="sheet-title">{title}</h2>    <!-- from opts.title -->
      <button class="sheet-close">X</button>  <!-- gray circle, SVG icon -->
    </div>
  </div>
  <div class="sheet-body">{contentHTML}</div> <!-- scrolls -->
</div>
```
- **Pass `title` separately** to `Sheet.open(opts)`. Do NOT put an `<h2>` in contentHTML.
- Sticky chrome keeps handle + title + close visible while body scrolls.
- Sheet has `border-top-left-radius: 22px`; chrome padding-top must clear that curve (currently 8px + 4px handle + 6px margin = enough).

### Sub-sheet (drill-down) pattern
Settings → tap "Budget" → opens EditValueSheet. Then user closes via Cancel or X.

**Problem we hit:** `Sheet.open()` closes any existing sheet first. So opening EditValueSheet closes Settings. On Save, Settings re-opens via callback. On **Cancel/X/Escape/scrim**, Settings stayed closed.

**Solution (LOCKED — don't undo this):**
- Sub-sheets accept `opts.onClose` and forward it to `Sheet.open({...onClose: opts.onClose})`
- Settings passes `onClose: function() { open(state, callbacks); }` on every sub-sheet launch
- Sheet.close() fires the onClose callback → Settings reopens on every close path

### FAB centering
Native `<button>+</button>` renders the + at the text baseline, not visual center. Either:
1. Use SVG `<path d="M12 5 L12 19 M5 12 L19 12">` (preferred)
2. Add `display: inline-flex; align-items: center; justify-content: center` to FAB
We did both for safety.

### Close button centering
Native `&times;` (×) has font-baseline offset that varies by font. Use SVG `<path d="M5 5 L15 15 M15 5 L5 15">` for pixel-perfect.

### RTL / Arabic
- Set `<html dir="rtl" lang="ar">` when locale = ar
- Use logical CSS properties: `margin-inline-start`, `padding-inline-end`, `text-align: start`
- Chevrons (`›`) need `transform: scaleX(-1)` in RTL
- Font fallback: `'SF Arabic', 'Segoe UI', 'Noto Naskh Arabic', sans-serif`
- All ~110 strings translated in `STRINGS.ar` in `src/i18n.js`
- Language toggle in Settings → Display

## Deploy recipe (use this, don't reinvent)

### To deploy v2 to GitHub Pages
1. `npm run build` in `v2/`
2. PowerShell script:
```powershell
$token = "ghp_..."           # fresh classic PAT, repo scope
$owner = "xCloudy75z"
$repoName = "rem-money"
$dist = "C:\Users\games\Desktop\Spending Tracker\v2\dist"
$headers = @{ Authorization = "Bearer $token"; "User-Agent" = "claude-deploy"; Accept = "application/vnd.github+json" }
$api = "https://api.github.com/repos/$owner/$repoName"

# Get current commit
$parentCommit = (Invoke-RestMethod -Uri "$api/git/ref/heads/main" -Headers $headers).object.sha

# Upload each file as a blob, collect into a tree
$files = Get-ChildItem -LiteralPath $dist -Recurse -File
$treeEntries = @()
foreach ($f in $files) {
  $rel = $f.FullName.Substring($dist.Length + 1) -replace '\\', '/'
  $b64 = [System.Convert]::ToBase64String([System.IO.File]::ReadAllBytes($f.FullName))
  $blob = Invoke-RestMethod -Uri "$api/git/blobs" -Method Post -Headers $headers `
    -Body (@{ content = $b64; encoding = "base64" } | ConvertTo-Json -Compress) `
    -ContentType "application/json"
  $treeEntries += @{ path = $rel; mode = "100644"; type = "blob"; sha = $blob.sha }
}
$tree = Invoke-RestMethod -Uri "$api/git/trees" -Method Post -Headers $headers `
  -Body (@{ tree = $treeEntries } | ConvertTo-Json -Depth 5) -ContentType "application/json"
$commit = Invoke-RestMethod -Uri "$api/git/commits" -Method Post -Headers $headers `
  -Body (@{ message = "Your commit message"; tree = $tree.sha; parents = @($parentCommit) } | ConvertTo-Json) `
  -ContentType "application/json"
Invoke-RestMethod -Uri "$api/git/refs/heads/main" -Method Patch -Headers $headers `
  -Body (@{ sha = $commit.sha } | ConvertTo-Json) -ContentType "application/json" | Out-Null

# Poll until built
for ($i = 0; $i -lt 60; $i++) {
  Start-Sleep -Seconds 5
  $latest = (Invoke-RestMethod -Uri "$api/pages/builds" -Headers $headers)[0]
  if ($latest.status -eq "built" -and $latest.commit -eq $commit.sha) { break }
}

# Verify URL serves new build
(Invoke-WebRequest -Uri "https://xcloudy75z.github.io/rem-money/" -UseBasicParsing).Content.Length
```

### **Per the verification-before-completion skill — ALWAYS fetch the live URL after deploy and grep for a specific marker from your change.** Don't claim "deployed" until you've seen the new bytes on the wire.

## Bug patterns hit this session (don't repeat)

### 1. PowerShell `Compress-Archive` uses Windows backslashes
**Symptom:** Files like `icons\apple-touch-icon-180.png` are uploaded as a single flat filename containing a backslash, not in an `icons/` folder. Netlify and most servers 404 these.
**Fix:** Use `[System.IO.Compression.ZipArchive]` directly and replace `\\` with `/` in entry paths.

### 2. Service worker over-aggressive navigation fallback
**Symptom:** `/logo-preview.html` returned cached `index.html`. Same for any new URL added later.
**Root cause:** SW's navigation handler did `caches.match('./index.html')` for ALL navigation requests, not just unknown ones.
**Fix in current sw.js:** Try the actual URL first, fall back to cached `./index.html` ONLY when offline AND no exact match.

### 3. iOS Safari white-icon on Add to Home Screen
**Symptom:** Installed PWA icon is a white square.
**Root cause:** SVG-only manifest; iOS doesn't render SVG apple-touch-icon reliably.
**Fix:** Ship PNG fallback at 180×180 referenced as `<link rel="apple-touch-icon">`.

### 4. Service worker caches old build during dev
**Symptom:** Code changes don't appear on reload.
**Fix:** In dev/preview, do:
```javascript
const keys = await caches.keys(); for (const k of keys) await caches.delete(k);
const regs = await navigator.serviceWorker.getRegistrations(); for (const r of regs) await r.unregister();
location.reload();
```

### 5. `preview_screenshot` times out unpredictably
**Symptom:** The MCP screenshot tool hangs and times out after 30s.
**Workaround:** Use `preview_eval` to query DOM state directly (`getBoundingClientRect`, `getComputedStyle`, etc.). It's more reliable than visual screenshots for verification.

### 6. Large `preview_eval` results are saved to file, not returned
**Symptom:** Calling eval with a big return value (>50K chars) returns an error pointing to a tool-result file path.
**Fix:** Read the file in chunks with offset/limit. Or split the eval into smaller batches that return shorter results.

### 7. iPhone sheet rendered behind status bar
**Symptom:** Sheet's "Settings" title is cut off at the top in PWA standalone mode.
**Root cause:** `max-height: 92vh` allows sheet to extend under status bar; sticky chrome anchored to sheet's top — which is under the status bar.
**Fix:** `max-height: calc(100vh - env(safe-area-inset-top) - 24px)` so the sheet stops below status bar entirely.

### 8. Fine-grained GitHub tokens have zero permissions by default
**Symptom:** Authenticated as user but `POST /user/repos` returns 403 Forbidden.
**Fix:** Either grant explicit Administration / Contents / Pages write permissions (for fine-grained) OR just use a classic PAT with `repo` scope. Classic is easier for a one-off deploy.

### 9. PowerShell `Invoke-WebRequest` warnings on IE engine
**Symptom:** "Windows PowerShell is in NonInteractive mode" errors when reading headers.
**Fix:** Use `-UseBasicParsing` flag. Always.

### 10. `cd` doesn't persist into background processes
**Symptom:** `cd X; Start-Process Y` runs Y in the original directory, not X.
**Fix:** Use `Start-Process -WorkingDirectory "X"`.
